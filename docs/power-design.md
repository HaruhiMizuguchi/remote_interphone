# 電源設計

> ここがかなり重要。

## 基本方針

Pico 経由でサーボに給電すると電流不足・電圧降下で不安定になる。
そのため **Pico WH とサーボを並列で 5V 電源につなぐ**。

## 正しい構成（採用）

```
5V電源
  ├─ Pico WH（VSYS）
  └─ サーボモータ（赤線）

GND（共通）
  ├─ Pico WH
  └─ サーボモータ
```

- Pico WH の GND とサーボの GND は共通
- Pico WH の GPIO 1本をサーボ信号線へ接続

## NG だった構成（不採用）

```
5V電源
  └─ Pico WH
       └─ サーボ（Pico の VBUS/3V3 から給電）
```

**不採用の理由:**

- サーボ動作時に大きな電流を引く
- Pico 内部のレギュレータ経由では電流が足りない
- 電圧降下で Pico 自体がリセットするリスク

## 電圧降下対策（推奨）

サーボ動作時の瞬間的な電圧降下を吸収するために、
サーボの電源ライン近くに**電解コンデンサ（470〜1000µF）**を入れると安定性が上がる。

```
5V ──┬── サーボ赤線
     │
     ├── コンデンサ（+）
     │
GND ─┴── コンデンサ（−）
```

これは必須ではないが、あると安心。

## 電源供給経路

```
5V ACアダプタ
  ↓
DCジャック-ネジ端子変換
  ├─ +5V → Pico WH（VSYS）
  ├─ +5V → サーボ（赤線）
  ├─ GND → Pico WH
  └─ GND → サーボ（黒/茶線）
```

USB ネジ端子アダプタが手元に無いため、**DCジャック方式**を採用。
