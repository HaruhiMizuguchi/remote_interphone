# 通信方式

## 最終目標

**外出先のスマホからインターホンの開錠ボタンを押せるようにする。**

## 段階的な通信構成

### ローカル（第2段階）

```
スマホ（同一 Wi-Fi）
  ↓ HTTP
Pico WH（簡易HTTPサーバ）
  ↓
サーボ動作
```

- Pico WH 上に簡易 HTTP サーバを立てる
- `/unlock` エンドポイントでサーボを動作させる

### リモート（第3段階）

外出先からアクセスするための方式候補:

#### 候補1: ポート開放

- 最も単純で Pico 単体で完結
- ルーターのポートフォワーディング設定が必要
- セキュリティリスクあり（秘密キー付き URL 等で最低限保護）

#### 候補2: Webhook / 中継サービス

- LINE Bot 等と連携しやすい
- 構成要素が増える（中継サーバが必要）
- セキュリティ面では直接公開より安全

### 目標イメージ

```
LINEで「開けて」
  ↓
Webhook / 中継サーバ
  ↓
Pico WH
  ↓
サーボ動作 → 開錠
```

## 優先順位

通信方式の最終決定はまだ先。
**まずはローカルで Pico → サーボが安定動作するところまで作るのが最優先。**
