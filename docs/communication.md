# 通信方式

## 最終目標

**外出先のスマホからインターホンの開錠ボタンを押せるようにする。**

## 採用方式: MQTT

マンションの共有回線（CGNAT）ではポート開放が不可能な場合が多いため、
**MQTT ブローカー経由**の方式を採用。

```
スマホ（モバイルデータ / Wi-Fi どこからでも）
  ↓ MQTT publish（WebSocket）
クラウド MQTT ブローカー
  ↑ MQTT subscribe（Pico から外向き接続）
Pico WH
  ↓
サーボ → 開錠
```

### なぜ MQTT か

| 比較項目 | ポート開放 | MQTT |
|----------|-----------|------|
| CGNAT 環境 | 不可 | 可能 |
| ルーター設定 | 必要 | 不要 |
| セキュリティ | 直接公開で高リスク | ブローカー認証 + シークレット |
| リアルタイム性 | 高い | 高い |
| Pico WH 対応 | ○ | ○（umqtt 標準） |

### セキュリティ

- MQTT ブローカーのユーザー名 / パスワード認証
- メッセージ本文に**共有シークレット**を含め、Pico 側で照合
- 本番環境では TLS（8883 / 8884）で通信を暗号化

## トピック設計

| トピック | 方向 | 内容 |
|----------|------|------|
| `interphone/cmd` | スマホ → Pico | 開錠コマンド（シークレット） |
| `interphone/status` | Pico → スマホ | 状態通知（online / unlocked / rejected） |

## ブローカー

### テスト用（認証なし）

| 項目 | 値 |
|------|----|
| ブローカー | `broker.hivemq.com` |
| MQTT ポート | 1883 |
| WebSocket ポート | 8000（ws://） |
| 認証 | なし |

### 本番用: HiveMQ Cloud（無料枠）

| 項目 | 値 |
|------|----|
| ブローカー | `xxxxx.s1.eu.hivemq.cloud` |
| MQTT ポート | 8883（TLS） |
| WebSocket ポート | 8884（WSS） |
| 認証 | ユーザー名 + パスワード |

## 将来の拡張

LINE Bot → クラウド関数 → MQTT publish の流れで
「LINE で『開けて』→ 開錠」の連携も自然に実現できる。
